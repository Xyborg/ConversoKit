# Changesets

Hi! This folder has been automatically generated by `@changesets/cli`, a tool for managing changesets and versioning in monorepos.

## How it works

1. **Add a changeset** when your PR changes a published package:
   ```bash
   pnpm changeset
   ```
   Pick the packages, the bump kind (patch / minor / major), and write a one-line summary. Commit the generated `.changeset/<random>.md` with your PR.

2. **On merge to `main`**, the CI workflow runs `changesets/action@v1`, which either:
   - opens a **Version Packages** PR aggregating all pending changesets and bumping versions + `CHANGELOG.md`, OR
   - if such a PR is already merged, publishes the bumped packages to npm.

3. **No manual `npm version` or `git tag` needed.**

See [the Changesets docs](https://github.com/changesets/changesets) for advanced usage.
